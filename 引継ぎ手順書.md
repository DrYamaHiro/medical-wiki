## 4. 初回セットアップ（新しいPCでの手順）

### 手順A: GitHubからクローンする場合（推奨）
```bash
git clone https://github.com/DrYamaHiro/medical-wiki.git
cd medical-wiki/medical-wiki
npm install
npm run start   # ブラウザで確認 → http://localhost:3000/medical-wiki/
```

### 手順B: このフォルダ（Google Drive）から復元する場合
```bash
cd CS-Temp-Dev_docusaurus/medical-wiki
npm install     # node_modules を再生成（数分かかります）
npm run start   # ブラウザで確認 → http://localhost:3000/medical-wiki/
```

## 6. Wiki内容の編集方法

### 方法A: GitHub上で直接編集（簡単）
1. https://github.com/DrYamaHiro/medical-wiki にアクセス
2. `medical-wiki/docs/` 配下の `.mdx` ファイルを開く
3. 鉛筆アイコンで編集 → Commit changes
4. 自動デプロイされる

### 方法B: ローカルで編集
1. `medical-wiki/docs/` 配下のMDXファイルをテキストエディタで開く
2. `{/* WIKI_EDIT_START */}` 〜 `{/* WIKI_EDIT_END */}` の間を編集
3. `npm run start` でプレビュー確認
4. git commit/push でデプロイ

> **重要**: WIKI_EDIT_START/END マーカーは削除しないでください。
> バージョンアップ時の自動更新スクリプトがこのマーカーを使います。

## 7. 診療データのバージョンアップ手順

新しいデータ（例: ver.2.0.2.0）が届いた場合：

```bash
# 1. 新しいデータを ver.2.0.2.0/output/ に配置する

# 2. 自動更新スクリプトを実行
cd CS-Temp-Dev_docusaurus/
node update_wiki.js ver.2.0.2.0

# → 以下を自動実行:
#   - git pull（GitHub上のWiki編集を取得）
#   - Wiki編集内容を退避（マーカー保護）
#   - MDXファイルを新バージョンで再生成
#   - Wiki編集内容を復元
#   - git commit/push（自動デプロイ）
```

## 8. GitHub Actions（自動デプロイ）の確認

デプロイが失敗した場合：
1. https://github.com/DrYamaHiro/medical-wiki/actions を開く
2. 最新のワークフロー実行を確認
3. エラーログを読む

よくある原因：
- MDXファイルの `<` `>` `{` `}` がエスケープされていない → `build_docs.js` の `escapeMdx()` 確認
- ビルドエラー → ローカルで `npm run build` を実行してエラー内容を確認

## 9. よくあるトラブルシューティング

| 問題 | 対処法 |
|---|---|
| `npm run start` でエラー | `npm install` を再実行 |
| ページが真っ白になる | ブラウザの開発者ツール(F12)でJSエラーを確認 |
| 公開URLが更新されない | GitHub Actionsのログを確認（5〜10分待つ） |
| MDXビルドエラー | エラーのファイル名を確認 → `<` を `&lt;` に修正 |
| `git push` が拒否される | `git pull` で最新を取得してからpush |
