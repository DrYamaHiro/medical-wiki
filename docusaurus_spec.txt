# 医療ナレッジベース構築システム仕様書 (Docusaurus版)

## 1. プロジェクト概要
本プロジェクトは、提供されたテキストデータ（診療テンプレート）をパースし、静的サイトジェネレータ「Docusaurus」を用いて、医療現場で使いやすい「3ペイン構成のWebサイト（Wiki）」を自動構築するものである。
API制限を回避するため、ローカル環境でテキストをMarkdown（MDX）ファイル群に一括変換し、最終的にGitHub Pages等で無料ホスティング・共同編集できる状態を目指す。

## 2. 開発環境・技術スタック
- 実行環境: Claude Code
- フロントエンド: Docusaurus v3 (React, Node.jsベース)
- パーサースクリプト: Python 3.10+
- ターゲット言語: MDX (Markdown + JSX)

## 3. ディレクトリ・レイアウト設計
Docusaurusの標準機能を活用し、以下のUIを実現する。
- 左サイドバー: カテゴリと疾患名の自動目次
- 中央エリア (MDXによる2カラム分割の左側): 診療テンプレート
- 右エリア (MDXによる2カラム分割の右側): 教科書的解説（共同編集エリア）

### MDXファイルの出力フォーマット設計
各疾患のファイル（例：`docs/010-Acute-URI/J15.9-Bacterial-Pneumonia.mdx`）は、以下の構造で出力すること。
Docusaurus内包のInfima CSSクラス（`row`, `col`）を使用して2カラムを実現する。

```mdx
---
id: j15-9-bacterial-pneumonia
title: "[J15.9] 細菌性肺炎 / 市中肺炎"
sidebar_position: 1
---

<div className="row">
  <div className="col col--7">
    ## 📝 診療テンプレート
    
    ### <S: 主訴・ROS>
    （ここにパースしたSのテキスト）

    ### <O: 所見・検査>
    （ここにパースしたOのテキスト）

    ### <Ns: 診察前対応>
    （ここにパースしたNsのテキスト）

    ### <A: 評価>
    （ここにパースしたAのテキスト）

    ### <P: 方針>
    （ここにパースしたPのテキスト）

    :::tip Dr.Advice
    （ここにパースしたDr.Adviceのテキストをリスト化）
    :::
  </div>
  
  <div className="col col--5">
    ## 📖 教科書的詳細解説
    :::info Wiki編集エリア
    この右側のエリアは、最新のガイドラインや病態生理、エビデンスを追記するためのスペースです。
    ページ下部の「このページを編集」から追記・修正を行ってください。
    :::
    
    ### 🔗 関連リンク・文献
    - [日本呼吸器学会 肺炎診療ガイドライン](https://www.jrs.or.jp/)
  </div>
</div>

4. テキストパース仕様 (Python Parser要件)
提供される ver.2.0.1.0.txt を読み込み、以下のルールで処理する。

================================================================ で囲まれた【No.XXX】を親カテゴリ（フォルダ）とする。

-------------------------------- で区切られたブロックを疾患ごとのデータとし、MDXファイルとして切り出す。

[ICD] 疾患名 を取得し、ファイル名およびフロントマターの title に設定する。

ファイル名やフォルダ名はURLセーフな英数字ケバブケース（例: 010-acute-uri）に変換すること。

<S:, <O:, <A:, <P:, ▶ Dr.Advice をそれぞれ抽出し、上記のMDXフォーマットに当てはめてファイルを出力する。

5. Claude Codeへの指示（実装ステップ）
Claude Codeに対して、以下の順序で実装を進めるよう指示する。

Step 1: Docusaurusプロジェクトの初期化。
Node.js環境にて npx create-docusaurus@latest medical-wiki classic を実行し、プロジェクトの雛形を作成せよ。

Step 2: Docusaurusの設定変更。
docusaurus.config.js を編集し、サイト名を「Medical Knowledge Wiki」等に変更。将来的なGitHub連携を見据え、editUrl の設定を残しておくこと。

Step 3: パーサースクリプト（build_docs.py）の作成。
要件定義「4」および「3」のMDXフォーマットに従い、ver.2.0.1.0.txt を読み込んで medical-wiki/docs/ ディレクトリ以下にフォルダとMDXファイルを自動生成するPythonスクリプトを作成せよ。

Step 4: パーサースクリプトの実行とビルド確認。
スクリプトを実行してドキュメントを生成し、不要なデフォルトドキュメント（hello.md等）を削除せよ。その後、npm run start（またはbuild）を行い、エラーが出ないか、意図した2カラムレイアウトで表示されるかを確認せよ。


### 次のアクション

1. お手元のPCに **Node.js** と **Python** がインストールされていることを確認してください（Claude Codeが動いているなら、おそらく入っているはずです）。
2. 先ほどの `ver.2.0.1.0.txt` と、保存した `docusaurus_spec.txt` を同じフォルダに置きます。
3. Claude Codeを起動し、以下のプロンプトを投げてください。

> 「カレントディレクトリにある `docusaurus_spec.txt` と `ver.2.0.1.0.txt` を読み込み、仕様書に記載されているStep 1から順に実装を開始してください。」

これを実行すると、Claude Codeが自動的にDocusaurusをセットアップし、テキストを解析して、見やすく構造化された美しいWebサイトのソースコード一式をあなたのPC上に作り上げます。

完成後、手元でプレビュー（`npm run start`等）して出来栄えを確認し、問題なければGitHubにアップロード（Push）するだけで、無料の医療Wikiサイトが全世界（または特定の仲間内）に公開されます！
